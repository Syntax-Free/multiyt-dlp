# CONTRIBUTING.md: The Intern’s Handbook

So, you want to touch the **Syntax Free Suite**? 

Before you submit a Pull Request (PR) and try to "fix" my masterpiece, understand the hierarchy: I am the **Architect**, and you are the **Unpaid Intern**. 

### 1. The Vibe Check (Coding Standards)
If your code is ugly, I will delete it. 
- **No Peasant UI:** If you’re adding a button, it better have a blur effect or a rounded corner. If it looks like Windows 95, stay away.
- **AI-Assistance is Encouraged:** If you didn't use an LLM to help write the fix, you're wasting your biological CPU cycles.
- **AI Vibe-check:** If Gemini deems the code is good, I'll merge the PR.

### 2. How to Submit a "Contribution"
1. **Find a Bug:** If you somehow find a bug in my *perfect* apps, go to step 2.
2. **Do the Labor:** Write the fix. Test it. Ensure it doesn't explode Syn’s RAM.
3. **Submit the PR:** Give it a title like "I did Tax's laundry" or "Fixing a minor oversight by the Genius."
4. **Wait:** I might look at it. I might not. I have a 15-hour nap scheduled.

*Note: By submitting a PR, you agree that I own this code now. I will take the credit. I will put it in a video. I will probably mock you during a shout-out.*

### 3. The Reward
By contributing, you earn the right to:
- Have your name in the GitHub contributors list (Clout).
- Not get roasted (maybe).
- If your PR is so good it makes me actually sit up in my chair, I might give you a shout-out.

*Note: If you want the 'Senior Dev' badge, the Discord perks, and the right to complain about bugs in the VIP lounge, go hit the Join button on YouTube. Don't expect a $5 title just because you fixed a semicolon.*
